Introduction

Many of ethslurp's options are available through the command line interface (cli). Other features, which do not fit easily into the cli, are accessible through a configuration file which is documented here. The file is called ethslurp.conf and is stored in ethslurp's cache folder.

EthSlurp's cache folder and the config file

Each time the program slurps block chain data, it stores that data in a binary format in a folder under the end user's $HOME folder called ~/.ethslurp. We store the configuration file (ethslurp.conf) in this same folder.

Configuration items are stored in the ethslurp.conf file on a single line (or, if more than one line, the line ends with the continuation mark ('\'). Additionally, configuration items are stored under an appropriate group header. The group header stands on its own line and is surrounded by square brackets: 

 #--------------------------------------------------------------
 # Contents of ethslurp.conf (lines starting with # are comments)
 #--------------------------------------------------------------
 [GROUP_HEADER_1]
 config_item1=A value stored on a single line
 config_item2=A series of values stored on multiple lines \
    any continued line of which ends with a backslash.

 [GROUP_HEADER_2]
 other_item=Another item

Individual configuration items are detailed below.

Configuration items

[SETTINGS] api_key
: This value is required in the current version of ethslurp because we use the etherscan.io apis. Each user must have thier own individual api key. We need to do this this because if every user used our api key, we would go over the per second limit on api calls pretty quickly. The software will not work until you provide your own key. In the future, we will read the block chain data directly from the block chain databases which will obviate the need for this parameter.

[SETTINGS] reset_rate
: By default ethslurp will only pull data from the block chain once every five minutes. This pararameter allows you to change the frequancy of update. The value is in seconds (therefore it defaults to 300). The lower limit on this value is 60 seconds (see above).

[EXPORT] fmt
: By default the results of the slurp are not formatted. In the special case of the -p (pretty print) command line option the results of the slurp are presented in a pretty print manner making them easier to read. You may, however, wish to have the data presented in numerous ways including tab seperated text files and/or comma seperated values that can be opened in MS Excel. This parameter allows you to control the format of the exported data. It is further detailed here.

Cached data

Ethslurp pulls data from the block chain under three conditions: (1) the first time you slurp an address, (2) once every five minutes, and (3) if you supply the '-s' flag. At all other times, ethslurp pulls the data from a locally stored cache. This is necessary to minimize the number of times the program hits the etherscan.io apis. You may change the refresh rate in a configuration file.

Planned improvements

The primary improvement we hope to provide before entering beta testing will be to pull the data directly from the Ethereum block chain as opposed to hitting up etherscan.io apis. This will greatly improve the speed of the software and remove the five minute caching requirement. Additionally, you will no longer need an etherscan.io API key.
