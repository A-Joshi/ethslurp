Export Formats

EthSlurp allows you to export the slurped blockchain data in any format you wish. While it does provide three pre-canned formats (json, txt, and csv), the number of available formats is limitless. This is accomplished through a feature we call 'display strings.'

Block chain data fields

Here's a list of all the fields in an Ethereum transaction: 

  blockHash, blockNumber, confirmations, contractAddress, cumulativeGasUsed,
  from, gas, gasPrice, gasUsed, hash, input, nonce, timeStamp, to,
  transactionIndex, value

Jumping directly to an example. The display string for exporting a tab seperated text file containing timeStamp, from, and value would be: 

 [{TIMESTAMP}]\t[{FROM}]\t[{VALUE}]\n                                       ~1

If instead you wanted blocknumber, to, gas, and value, you would use: 

 [{BLOCKNUMBER}]\t[{TO}]\t[{GASUSED}]\t[{VALUE}]\n                          ~2

If you wanted comma seperated values so you could open the data in MS Excel, you would use: 

 ["{BLOCKNUMBER}"],["{TO}"],["{GASUSED}"],["{VALUE}"]\n                     ~3

If you wanted something human readable, you would use: 

 [Num:\t{BLOCKNUMBER}\n][To:\t{TO}\n][Gas:\t{GASUSED}\n][Value:\t{VALUE}\n] ~4

A couple of things to note about display strings:

  • The field name in the string is the same as the name of the data field except it is in UPPERCASE;
  • Each field is surrounded directly with squiggly brackets. A field name will not be recognized if it is not in UPPERCASE and surrounded directly with squiggly brackets;
  • Each field is further surrounded with square brackets. Again a field will not be recognized without its square brackets, but this time the square brackets do not need to be touching the squiggly brackets;
  • If a field's value is empty, the entire [...] string will evaluate to empty.

Furthering the example, if the data for a transaction looked like this: 

  "blockHash": "0xf61cc7b7d3ef647bcb6218a0.....a17b10364e255508",
  "blockNumber": "1491831",
  "confirmations": "7",
  "contractAddress": "",
  "cumulativeGasUsed": "408675",
  "from": "0x0ec97fcf4dd1a68a...",
  "gas": "121000",
  "gasPrice": "20000000000",
  "gasUsed": "74091",
  "hash": "0xa2b399a267ad13c2142e7fa388864ce7820....1616b81bb16a6c0a2e2",
  "input": "0x",
  "nonce": "3",
  "timeStamp": "1462885596",
  "to": "0xbb9bc244d798123...",
  "transactionIndex": "6",
  "value": "1000000000000000000"

then the above display strings would render as follows: 

 [{TIMESTAMP}]\t[{FROM}]\t[{VALUE}]\n                                       ~1

   1462885596    0x0ec97fcf4dd1a68a...    1000000000000000000

 [{BLOCKNUMBER}]\t[{TO}]\t[{GASUSED}]\t[{VALUE}]\n                          ~2

   1491831        0xbb9bc244d798123...    74091    1000000000000000000

 ["{BLOCKNUMBER}"],["{TO}"],["{GASUSED}"],["{VALUE}"]\n                     ~3

   "1491831","0xbb9bc244d798123...","74091","1000000000000000000"

 [Num:\t{BLOCKNUMBER}\n][To:\t{TO}\n][Gas:\t{GASUSED}\n][Value:\t{VALUE}\n] ~4

   Num:    1491831
   To:     0xbb9bc244d798123...
   Gas:    74091
   Value:  1000000000000000000

Note that the display string applies to each record as it is exported.

What can you do with display strings?

Everyone talks about the shared, immortal, immutable, infinitely accessible, shared ledger that the block chain enables. "It will make all of our lives life easier," they say. Until now, getting at this data was hard. We know. We created this tool to help ourselves get at the data in an easier way. It's helped us immensely.

We hope it helps you as well.
